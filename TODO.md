## Блок 2 — Direct Movement (левый стик)

### Как работает движение в Vanilla
Клиент шлёт `wdgmsg("click", screenCoord, worldCoord, button, mods, ...)` в `MapView`.  
Сервер отвечает `OD_LINBEG` / `OD_LINSTEP` дельтами → `LinMove`. Персонаж движется к целевой точке, может выйти за неё максимум на `MAXOVER = 0.5` сек (см. `LinMove.java:34`).

### Расчёт дистанции клика

Скорость персонажа в HnH **фиксирована** на уровне сервера — клик в любую точку даёт одинаковую скорость бега.  
Поэтому дистанция клика — константа, подобранная экспериментально.

| Параметр | Значение |
|---|---|
| Интервал кликов | 150–200 мс |
| Целевая дистанция | **~2 тайла** вперёд по вектору стика |
| Формат координат | `posres = 2^-10 × 11` на ось (`OCache.java:59`) |

**Правило остановки:** при отпускании стика персонаж дойдёт до последней точки и остановится. При 2 тайла — чисто. При слишком малой (< 0.5 тайла) — сервер не начнёт новый LINBEG → рывки. При слишком большой (> 3–4 тайла) — персонаж «уплывает» после отпускания.

### Задачи
- [ ] `DirectMovement` — класс, получает вектор `(dx, dy)` от стика каждые 150 мс
  - Нормировать вектор, учесть угол камеры (`MapView.camera.angle()`)
  - Вычислить целевую экранную точку: `player_screen_pos + dir_rotated * click_dist_px`
  - Сконвертировать в мировые координаты через `MapView.screenxf()` / существующий ray-cast
  - Отправить через `MapView.wdgmsg("click", ...)`
- [ ] Деадзона стика: `|stick| < 0.15` → не отправлять клик (не сбрасывать движение)
- [ ] При `|stick| == 0` (отпускание) — прекратить отправку; персонаж сам остановится
- [ ] Tune-параметры вынести в `GamepadConfig` (дистанция, интервал)
- [ ] Убедиться, что synthetic-клики не конфликтуют с реальными мышиными (флаг `from_gamepad`)

## Блок 3 — Камера

### 3.1 Снятие углового блока (приоритет)
`FollowCam` в `MapView.java:195–203` блокирует свободный наклон камеры двумя ограничениями:
- `maxang = PI/2 - 0.1` — нельзя смотреть строго сверху вниз
- `if(dist(telev) < mindist) telev = fe` — при малых углах отклоняет изменение elevation,  
  фактически блокируя вид «с горизонта»

- [ ] Убрать или вынести в конфиг `maxang` → разрешить полный диапазон `[epsilon, PI/2]`
- [ ] Убрать блок `dist(telev) < mindist` для elevation (оставить только для zoom)
- [ ] Добавить нижний предел elevation ~`PI/16` чтобы камера не уходила под землю
- [ ] L2+RS-Y управляет `telev` напрямую без этих блоков (уже существующая интерполяция подхватит)

### 3.2 Управление камерой правым стиком

#### Без L2 — правый стик = мышь
- [ ] RS `(rx, ry)` → синтетические `MouseMoveEvent` в `UI` (дельта позиции)
- [ ] Чувствительность настраивается отдельно от движения
- [ ] Реальная мышь не отключается — оба источника суммируются

#### С зажатым L2 — правый стик = вращение/наклон камеры
- [ ] L2 как модификатор: `GamepadState.l2_held`
- [ ] RS-X → `camera.tangl += rx * sensitivity` (аналог перетаскивания средней кнопкой)
- [ ] RS-Y → `camera.telev += ry * sensitivity`, зажать в `[PI/8, maxang]`
- [ ] Использовать существующую анимацию (`FollowCam.tick()` уже интерполирует `angl`/`elev`)
- [ ] При отпускании L2 — плавно отдать RS обратно под мышь (без рывка)

## Блок 4 — Умный таргет (R1)

R1 = **умный правый клик** на лучший объект в конусе перед игроком.  
Правый клик в vanilla HnH контекстно-зависим: на лут → подобрать, на станок → открыть меню, на моба → атаковать/флаер.  
Если появляется FlowerMenu от сервера — оно навигируется RS + A.

### Алгоритм поиска в конусе

```
cone_angle = 60° (±30° от вектора взгляда персонажа)
cone_depth  = 5–8 тайлов
```

Вектор взгляда = направление последнего движения или угол камеры, если стоит.

### Режимы таргета

#### Обычный режим

| Приоритет | Категория | Ресурс-путь |
|---|---|---|
| 1 | **Двери / врата** | `gfx/terobjs/gate`, `gfx/terobjs/door` |
| 2 | **Лут / предметы на земле** | `gfx/terobjs/items/` |
| 3 | **Мешки внутри повозки** | bag-объекты рядом с vehicle-gob |
| 4 | **Станки / рабочие столы** | `gfx/terobjs/` (craft-подтипы) |
| 5 | **Контейнеры** | `gfx/terobjs/chest`, `gfx/terobjs/basket` |
| 6 | **Животные / НПС** | `gfx/kritter/`, `gfx/borka/` |
| 7 | **Ресурсы** (деревья, камни) | `gfx/trees/`, `gfx/stones/` |
| — | **Повозка сама по себе** | игнорируется если рядом есть мешки |

#### Боевой режим (отдельный флаг, переключатель L1)

| Приоритет | Категория |
|---|---|
| 1 | Текущий lock-цель (если есть) |
| 2 | Ближайший агрессивный моб |
| 3 | Ближайший нейтральный моб / животное |
| 4 | Игрок (PvP) |

В боевом режиме обычный лут/станки убираются из конуса — только цели для атаки.  
Визуальный индикатор: цвет курсора / иконка в HUD меняется.

### Механика повозки (матрёшка)
Повозка — самостоятельный Gob. При размещении предметов в ней на карте появляются  
**отдельные Gob-мешки** в пределах 1–2 тайлов от позиции повозки.  
Кликая повозку — цепляешь её. Кликая мешок — забираешь содержимое.

Поведение SmartTarget:
- Мешки рядом с vehicle-gob → приоритет 3 (выше самой повозки)
- Если несколько мешков → `RadialPicker` со списком мешков
- Сама повозка попадает в таргет только если мешков нет рядом

### Задачи
- [ ] `SmartTarget` — сканирует `glob.oc`, фильтрует объекты в конусе
  - Ресурс-имя: `Gob.getattr(ResDrawable.class).res.get().name`
  - Угол от игрока до объекта vs. вектор взгляда
- [ ] Определение «вектора взгляда»:
  - Если стик двигался < 1 сек назад — последнее направление движения (хранить постоянно)
  - Иначе — XY-проекция вектора «от камеры к персонажу»
- [ ] Подсветка таргета: `Overlay` на выбранном Gob (аналог существующих оверлеев)
- [ ] Флаг боевого режима: `CombatMode.active`, переключается по L1
- [ ] Визуальная индикация боевого режима в HUD

#### R1 одиночное нажатие — умный правый клик
- [ ] Найти лучший Gob в конусе по таблице приоритетов (с учётом режима)
- [ ] Один объект → правый клик: `wdgmsg("click", ..., 3, ...)`
- [ ] Несколько в одном приоритетном классе → `RadialPicker`
- [ ] Если появился FlowerMenu от сервера → он навигируется RS + A (не блокируем vanilla-флаер)

#### R1 удержание — радиальное окно ближайших объектов

## Открытые вопросы

1. **Боевой режим — автоактивация?** Нужно ли включать боевой режим автоматически при агро (моб атакует игрока), или только вручную через L1?

2. **Вектор взгляда при простое**: Хранить последнее направление движения бессрочно, или после N секунд простоя брать вектор «от камеры к персонажу»?

3. **D-pad и инвентарь**: Как переключать фокус между несколькими открытыми окнами инвентаря — например, сундук + рюкзак одновременно? Нужна ли кнопка для смены фокуса?

4. **Библиотека геймпада**: jinput не обновлялся с 2017. Есть ли предпочтения? SDL2/JNA — лучше поддержка PS/Xbox геймпадов и rumble, но сложнее в сборке.

5. **RadialPicker — максимум объектов**: При удержании R1 в густом лесу могут попасть десятки деревьев. Ограничить до 8? Или показывать только топ-N по каждой категории?
